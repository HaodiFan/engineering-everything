# Codex Tools Contract（工具契约）

本文件给 bootloader 提供最小工具边界。具体项目仍以用户仓库的 AGENTS.md、开发者指令和当前工具可用性为准。

## 默认策略

- 搜索文件优先使用 `rg` / `rg --files`。
- 手工编辑文件优先使用 `apply_patch`。
- 复杂重构先确认验证网，再改行为相关文件。
- 用户要求多 agent 时，拆成边界清晰、互不重叠的任务。
- 涉及最新事实、法律、价格、API 文档或外部规则时，先查证再下结论。

## 停止条件

- 关键事实无法从本地、官方文档或用户输入确认。
- 设计需要 owner 决策，例如 license、发布、合规边界、公开 API 破坏性变更。
- 验证命令失败，且失败原因未被解释或修复。

## 收尾要求

收尾必须说明变更文件、已运行验证、未运行检查及原因、剩余风险。发布或版本建议必须区分“已发布版本”和“建议下一个版本”。
